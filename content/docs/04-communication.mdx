## 一、整體架構總覽

### 網路拓樸

```
手機瀏覽器
  ├─ WebSocket ──────────→ 地面站 ESP32（192.168.4.1）  遙測數據（5Hz）
  ├─ HTTP POST ──────────→ 地面站 ESP32                  航點上傳
  └─ MJPEG <img> ────────→ ROV ESP32（192.168.4.100）   影像直連（不經地面站）

地面站 ESP32（純 AP，192.168.4.1）
  └─ ESP-NOW ↕ ROV ESP32（STA，192.168.4.100）
```

### 各裝置職責

| 裝置 | 負責 | 不負責 |
|------|------|--------|
| 地面站 ESP32 | SoftAP、WebSocket 推遙測、HTTP 接收航點、ESP-NOW 轉發、Xbox USB HID | 影像（完全不碰） |
| ROV ESP32 | 馬達控制、感測器讀取、ESP-NOW 收發、MJPEG HTTP Server | — |
| 手機瀏覽器 | 地圖、遙測顯示、航點設置、影像顯示 | — |
| Xbox 手把 | 有線 USB HID 接地面站 ESP32，輸出搖桿與按鍵值 | — |

## 二、Wi-Fi 配置

### 模式選擇

- **地面站**：純 AP 模式（`WIFI_MODE_AP`），不使用 APSTA，避免 STA home channel 強制覆蓋 AP channel 的陷阱
- **ROV**：純 STA 模式（`WIFI_MODE_STA`），連入地面站 AP
- ESP-NOW 在 AP/STA 模式下均可正常運作，頻道自動跟隨 Wi-Fi

### 地面站初始化

```cpp
#define WIFI_CHANNEL 1
#define AP_SSID      "ROV_GS"
#define AP_PASS      "rov12345"

void setupWifi_GS() {
    WiFi.mode(WIFI_MODE_AP);
    WiFi.softAP(AP_SSID, AP_PASS, WIFI_CHANNEL);
    esp_wifi_set_ps(WIFI_PS_NONE);  // 關閉省電，防漏包

    Serial.print("GS AP MAC: ");
    Serial.println(WiFi.softAPmacAddress());  // 記下這個，填入 ROV peer_addr
}
```

### ROV 初始化

```cpp
void setupWifi_ROV() {
    WiFi.mode(WIFI_STA);
    WiFi.config(IPAddress(192,168,4,100),   // 固定 IP，手機 <img src> 不會失效
                IPAddress(192,168,4,1),     // Gateway = 地面站
                IPAddress(255,255,255,0));
    WiFi.begin(AP_SSID, AP_PASS);
    esp_wifi_set_ps(WIFI_PS_NONE);

    Serial.print("ROV STA MAC: ");
    Serial.println(WiFi.macAddress());  // 記下這個，填入地面站 peer_addr
}
```

## 三、ESP-NOW 配置

### MAC 對應規則（最容易踩坑）

ESP32 的 AP 介面與 STA 介面有不同 MAC（AP MAC = Base MAC 最後 byte + 1）。

| 端 | `ifidx` | `peer_addr` 填入 |
|----|---------|-----------------|
| 地面站新增 ROV 為 Peer | `WIFI_IF_AP` | ROV 的 **STA MAC** |
| ROV 新增地面站為 Peer | `WIFI_IF_STA` | 地面站的 **AP MAC** |

`peer.channel = 0` 表示跟隨本機當前頻道，最安全，不需硬填頻道號。

若頻道不匹配，ESP-NOW 會**靜默失敗**（send callback 回報 `ESP_NOW_SEND_FAIL`，無明確錯誤訊息）。

### 地面站 ESP-NOW 初始化

```cpp
uint8_t rov_sta_mac[] = {0xXX, 0xXX, 0xXX, 0xXX, 0xXX, 0xXX};  // ROV 的 STA MAC

void setupESPNOW_GS() {
    esp_now_init();
    esp_now_register_recv_cb(onDataRecv_GS);
    esp_now_register_send_cb(onDataSent_GS);

    esp_now_peer_info_t peer = {};
    memcpy(peer.peer_addr, rov_sta_mac, 6);
    peer.channel = 0;
    peer.ifidx   = WIFI_IF_AP;   // 地面站用 AP 介面
    peer.encrypt = false;
    esp_now_add_peer(&peer);
}
```

### ROV ESP-NOW 初始化

```cpp
uint8_t gs_ap_mac[] = {0xXX, 0xXX, 0xXX, 0xXX, 0xXX, 0xXX};  // 地面站的 AP MAC

void setupESPNOW_ROV() {
    esp_now_init();
    esp_now_register_recv_cb(onDataRecv_ROV);
    esp_now_register_send_cb(onDataSent_ROV);

    esp_now_peer_info_t peer = {};
    memcpy(peer.peer_addr, gs_ap_mac, 6);
    peer.channel = 0;
    peer.ifidx   = WIFI_IF_STA;  // ROV 用 STA 介面
    peer.encrypt = false;
    esp_now_add_peer(&peer);
}
```

### ESP-NOW 資料結構

```cpp
// 地面站 → ROV（控制指令）
struct ControlPacket {
    int16_t  leftMotor;     // -1023 ～ +1023
    int16_t  rightMotor;
    int16_t  vertMotor;
    bool     ledOn;
    bool     emergencyStop;
    bool     autoMode;
    uint8_t  streamMode;    // 0 = 純串流，1 = 串流 + SD 錄影
    bool     takePhoto;     // true = 觸發單張拍照（LB 邊緣觸發，純串流模式下有效）
    uint8_t  msgType;       // 0=控制, 1=航點塊, 2=ACK
} __attribute__((packed));

// ROV → 地面站（遙測）
struct TelemetryPacket {
    float    lat;
    float    lng;
    float    depthM;
    float    currentA;
    float    powerW;
    uint8_t  batPct;
    int8_t   rssi;
    uint8_t  streamMode;       // ROV 確認回傳
    uint8_t  navWaypointIdx;   // 自動導航目標航點索引（0xFF = 未導航）
    float    navDistanceM;     // 距目標航點距離（公尺）
    bool     photoAck;         // 拍照完成確認
    uint8_t  msgType;          // 3=遙測
} __attribute__((packed));
```

## 四、WebSocket 遙測

### 地面站 WebSocket Server

```cpp
#include <ESPAsyncWebServer.h>  // ESP32Async fork
#include <ArduinoJson.h>

AsyncWebServer server(80);
AsyncWebSocket ws("/ws");

void broadcastTelemetry(TelemetryPacket& pkt) {
    JsonDocument doc;
    doc["lat"]       = pkt.lat;
    doc["lng"]       = pkt.lng;
    doc["depth"]     = pkt.depthM;
    doc["current"]   = pkt.currentA;
    doc["power"]     = pkt.powerW;
    doc["bat"]       = pkt.batPct;
    doc["rssi"]      = pkt.rssi;
    doc["streamMode"]= pkt.streamMode;
    doc["navWpIdx"]  = pkt.navWaypointIdx;
    doc["navDistM"]  = pkt.navDistanceM;
    doc["photoAck"]  = pkt.photoAck;

    size_t len = measureJson(doc);
    auto *buf = ws.makeBuffer(len);
    if (buf) {
        serializeJson(doc, buf->get(), len + 1);
        ws.textAll(buf);
    }
}
```

推送頻率：**5–10Hz（200–100ms 間隔）**，超過 20Hz 會擠壓 ESP-NOW 無線電時間。

### 手機瀏覽器 WebSocket 自動重連

```javascript
let ws, reconnectDelay = 1000;
let photoToastTimer = null;

function connectWS() {
    ws = new WebSocket('ws://192.168.4.1/ws');
    ws.onmessage = (e) => {
        const d = JSON.parse(e.data);
        updateTelemetry(d);
        updateStreamModeBadge(d.streamMode);
        updateRSSIWarning(d.rssi);
        updateNavStatus(d.navWpIdx, d.navDistM);
        if (d.photoAck) showPhotoToast();
        reconnectDelay = 1000;
    };
    ws.onclose = () => {
        setTimeout(connectWS, reconnectDelay);
        reconnectDelay = Math.min(reconnectDelay * 2, 30000);
    };
}
connectWS();
```

## 五、航點上傳

### 採用 HTTP POST（非 WebSocket）

航點上傳是一次性操作，HTTP POST 有明確狀態碼回應（200/400/500），比 WebSocket fire-and-forget 更可靠。

### 地面站端接收

```cpp
server.on("/api/waypoints", HTTP_POST,
    [](AsyncWebServerRequest *req) {},
    nullptr,
    [](AsyncWebServerRequest *req, uint8_t *data, size_t len, size_t index, size_t total) {
        JsonDocument doc;
        deserializeJson(doc, data, len);
        JsonArray wps = doc["waypoints"];

        waypointCount = 0;
        for (JsonObject wp : wps) {
            waypoints[waypointCount].lat   = wp["lat"];
            waypoints[waypointCount].lng   = wp["lng"];
            waypoints[waypointCount].order = wp["order"];
            waypointCount++;
        }
        sendWaypointsToROV();
        req->send(200, "application/json", "{\"status\":\"ok\"}");
    }
);
```

### ESP-NOW 分塊傳輸

ESP-NOW 封包上限 250 bytes，每塊可裝 **14 個航點**（14 × 17 bytes = 238 bytes + 5 bytes 標頭）。50 個航點只需 4 個封包。

```cpp
struct WaypointChunk {
    uint8_t  msgType;       // 1 = 航點塊
    uint8_t  seqNum;
    uint8_t  totalChunks;
    uint8_t  chunkIndex;
    uint8_t  wpCount;
    struct { double lat; double lng; uint8_t order; } wps[14];
} __attribute__((packed));

void sendWaypointsToROV() {
    int totalChunks = (waypointCount + 13) / 14;
    for (int c = 0; c < totalChunks; c++) {
        WaypointChunk chunk;
        chunk.msgType     = 1;
        chunk.totalChunks = totalChunks;
        chunk.chunkIndex  = c;
        chunk.wpCount     = min(14, waypointCount - c * 14);
        for (int i = 0; i < chunk.wpCount; i++) {
            int idx = c * 14 + i;
            chunk.wps[i].lat   = waypoints[idx].lat;
            chunk.wps[i].lng   = waypoints[idx].lng;
            chunk.wps[i].order = waypoints[idx].order;
        }
        esp_now_send(rov_sta_mac, (uint8_t*)&chunk, sizeof(chunk));
        delay(50);
    }
}
```

## 六、地圖與離線策略

### Leaflet.js + 雙重降級

Leaflet.js 存於 ESP32 LittleFS（gzip 後約 45KB），地圖磁磚走手機行動數據。**不使用 Google Maps**：Google Maps JS API 需從 Google CDN 載入，手機連上 ESP32 AP 後 Android 多數裝置會停止使用行動數據，無法保證可靠載入。

```javascript
const tileLayer = L.tileLayer(
    'https://tile.openstreetmap.org/{z}/{x}/{y}.png',
    { maxZoom: 18 }
);
tileLayer.on('tileerror', () => switchToCanvasGrid());
```

### LittleFS 檔案架構

```
data/www/
├── index.html           ~5 KB
├── style.css.gz         ~3 KB
├── app.js.gz            ~8 KB
├── leaflet.js.gz        ~45 KB
├── leaflet.css.gz       ~4 KB
└── images/
    ├── marker-icon.png  ~1 KB
    └── marker-shadow.png ~1 KB
─────────────────────────────────
總計 ~70 KB（gzip），LittleFS 1.5MB 分區輕鬆容納
```

## 七、地面站 setup() 初始化順序

```cpp
void setup() {
    Serial.begin(115200);
    setupWifi_GS();       // SoftAP 啟動（必須在 ESP-NOW 之前）
    setupESPNOW_GS();     // ESP-NOW 初始化
    LittleFS.begin();
    server.serveStatic("/", LittleFS, "/www/")
          .setDefaultFile("index.html")
          .setCacheControl("max-age=600");
    ws.onEvent(onWSEvent);
    server.addHandler(&ws);
    server.begin();
    setupXbox();          // USB HID
}

void loop() {
    ws.cleanupClients();
    readXbox();           // 讀搖桿並發 ESP-NOW（100Hz）
}
```

## 八、常見踩坑清單

| 問題 | 原因 | 解法 |
|------|------|------|
| ESP-NOW 靜默失敗 | ifidx 或 peer MAC 填錯 | GS 填 ROV 的 STA MAC + `WIFI_IF_AP`；ROV 填 GS 的 AP MAC + `WIFI_IF_STA` |
| 頻道不匹配導致失敗 | APSTA 模式 STA 頻道覆蓋 AP 頻道 | GS 改用純 AP 模式（已修正） |
| 手機看不到影像 | ROV IP 被 DHCP 動態分配 | ROV 設定固定 IP 192.168.4.100 |
| ESP-NOW 漏包 | Wi-Fi 省電模式 | 兩端均設 `esp_wifi_set_ps(WIFI_PS_NONE)` |
| BLE + ESP-NOW 干擾 | 2.4GHz 無線電資源競爭 | 改用有線 USB HID Xbox |
| WebSocket 斷線後無法恢復 | 未實作自動重連 | 瀏覽器端實作指數退避重連 |
| Google Maps 無法載入 | 手機連 ESP32 AP 後 Android 停用行動數據 | 改用 Leaflet.js + Canvas 降級 |
