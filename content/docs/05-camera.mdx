## 二、影像傳輸方案：手機直連 ROV

ESP32 SoftAP 預設不啟用 AP Isolation，同一 AP 下的 STA 之間可直接互通（Layer-2 硬體轉發）。手機（192.168.4.x）可直接存取 ROV（192.168.4.100）的 HTTP Server，地面站零 CPU 負擔。

```html
<!-- 手機網頁中只需這一行 -->
<img src="http://192.168.4.100:80/stream"
     style="width:100%; max-height:40vh; object-fit:contain;">
```

### 各方案比較

| 評估項目 | 方案 A：地面站中繼 | **方案 B：直連 ROV（採用）** | 方案 C：ESP-NOW |
|---------|-----------------|--------------------------|--------------|
| 地面站 CPU | 極重 | **零** | 極重 |
| Wi-Fi 頻寬 | 2× | **1×** | 不可行 |
| 延遲 | +50–150ms | **最低（單跳）** | 數百 ms |
| VGA@15fps | 勉強 | **輕鬆** | 完全不可行 |

方案 C 不可行原因：ESP-NOW unicast 實測吞吐約 350KB/s，VGA JPEG 每幀 30–50KB 需 120–200 個封包，理論幀率不到 5fps，且任一封包遺失整幀作廢。

## 三、串流模式總表

| 模式 | 觸發 | 串流解析度 | 幀率 | Wi-Fi 串流 | SD 錄影 | LB 手動拍照（固定 720p） |
|------|------|-----------|------|-----------|---------|------------------------|
| 0 純串流（預設） | 開機 / RB 切回 | RSSI 動態 | 15–20 FPS | ✅ | — | ✅ 可用 |
| 1 串流 + SD 錄影 | RB 切換 | RSSI 動態 | 15–20 FPS | ✅ | ✅ | ❌ 鎖定 |

RB 鍵邊緣觸發，在模式 0 / 1 之間切換。ROV 切換後經 TelemetryPacket 回傳確認。

### 解析度分工

| 功能 | 解析度 | 決定依據 |
|------|--------|---------|
| 串流（兩種模式均適用） | 800×600 / 640×480 | RSSI 動態切換 |
| LB 手動拍照 | 固定 720p（1280×720） | 與 RSSI 無關，固定最高畫質 |

### 模式 1 拍照鎖定原因

SD 卡連續寫幀幾乎佔滿 SDMMC 頻寬（最高約 2MB/s），插入 720p 單幀寫入會造成幀緩衝溢出，破壞 .avi 時序。

## 四、RSSI 降級邏輯

| RSSI | 動作 |
|------|------|
| > −60 dBm | SVGA 800×600，15–20 FPS |
| −60 ～ −75 dBm | VGA 640×480，15 FPS |
| < −75 dBm | 暫停串流，等待重連 |

```cpp
void applyStreamQuality(int8_t rssi) {
    sensor_t *s = esp_camera_sensor_get();
    if (rssi > -60) {
        s->set_framesize(s, FRAMESIZE_SVGA);  // 800×600
        s->set_quality(s, 10);
    } else if (rssi > -75) {
        s->set_framesize(s, FRAMESIZE_VGA);   // 640×480
        s->set_quality(s, 12);
    } else {
        streamActive = false;  // 暫停串流
    }
}
```

## 六、手動拍照（LB 鍵，純串流模式下有效）

```
LB 鍵邊緣觸發 → takePhoto = true（單次脈衝）
    ↓ ControlPacket 傳至 ROV

模式 0：暫停串流 → 切換 720p → 拍照存 SD → 恢復串流解析度
模式 1：忽略（SD 正在錄影，鎖定）
```

### ROV 端實作

```cpp
void takePhotoAndSave() {
    // 1. 暫停串流
    streamActive = false;
    vTaskDelay(50);  // 等待當前幀傳送完畢

    // 2. 切換感測器至 720p
    sensor_t *s = esp_camera_sensor_get();
    framesize_t prevSize = s->status.framesize;
    s->set_framesize(s, FRAMESIZE_HD);  // 1280×720

    // 3. 擷取並存 SD
    camera_fb_t *fb = esp_camera_fb_get();
    if (fb) {
        char filename[32];
        sprintf(filename, "/photo_%lu.jpg", millis());
        File f = SD_MMC.open(filename, FILE_WRITE);
        if (f) { f.write(fb->buf, fb->len); f.close(); }
        esp_camera_fb_return(fb);
    }

    // 4. 切回串流解析度並恢復
    s->set_framesize(s, prevSize);
    streamActive = true;

    // 5. 回傳拍照確認（下一個遙測週期帶出）
    photoAckPending = true;
}
```

手機串流約停頓 0.3–0.5 秒。拍照完成後 `takePhoto` 自動歸 false。

## 七、SD 錄影（AVI MJPEG 格式）

### 錄影格式要求

錄影寫入 `.avi` 檔案，必須遵循完整的 **RIFF/AVI 容器格式**，否則播放器無法解析。直接把 JPEG 幀 raw bytes 連續寫入然後命名成 `.avi` 是**不可播放的**。

### AVI MJPEG 容器結構

```
RIFF 'AVI '
  ├── LIST 'hdrl'          ← AVI 主標頭
  │     ├── 'avih'         ← 主標頭（幀率、幀寬高、串流數）
  │     └── LIST 'strl'    ← 串流標頭
  │           ├── 'strh'   ← 串流標頭（fccType='vids', fccHandler='MJPG'）
  │           └── 'strf'   ← 串流格式（BITMAPINFOHEADER）
  ├── LIST 'movi'          ← 影像資料區
  │     ├── '00dc' + JPEG 幀 1
  │     ├── '00dc' + JPEG 幀 2
  │     ├── ...
  │     └── '00dc' + JPEG 幀 N
  └── 'idx1'               ← 索引區（每幀的偏移量與大小）
```

### 實作流程

1. **開始錄影**：寫入 RIFF 標頭 + hdrl（幀率設 15、解析度 640×480、codec MJPG），此時總大小欄位先填 0
2. **每幀寫入**：每個 JPEG 幀前加 `'00dc'` chunk header（4 bytes tag + 4 bytes size），同時記錄該幀在檔案中的偏移量與大小（存於 RAM 陣列，供 idx1 使用）
3. **結束錄影**：寫入 `idx1` 索引區，回頭更新 RIFF 標頭的總大小欄位，關閉檔案

### 關鍵程式碼框架

```cpp
File aviFile;
uint32_t frameCount = 0;
uint32_t moviOffset = 0;

// 索引暫存（每幀 16 bytes：4 tag + 4 flags + 4 offset + 4 size）
// 15fps × 60s × 30min ≈ 27000 幀 × 16 bytes ≈ 432KB，需存在 PSRAM
struct AviIdx {
    uint32_t offset;
    uint32_t size;
};
AviIdx* idxBuffer;  // ps_malloc() 分配

void startRecording() {
    char filename[32];
    sprintf(filename, "/rec_%lu.avi", millis());
    aviFile = SD_MMC.open(filename, FILE_WRITE);

    // 分配索引緩衝（PSRAM）
    idxBuffer = (AviIdx*)ps_malloc(sizeof(AviIdx) * 30000);
    frameCount = 0;

    // 寫入 RIFF + hdrl（固定結構，約 224 bytes）
    writeAviHeader(aviFile, 640, 480, 15);  // 寬、高、幀率
    moviOffset = aviFile.position();

    // 寫入 LIST 'movi' 標頭（大小先填 0，結束時回補）
    aviFile.write("LIST", 4);
    uint32_t placeholder = 0;
    aviFile.write((uint8_t*)&placeholder, 4);
    aviFile.write("movi", 4);
}

void writeFrame(camera_fb_t *fb) {
    if (!aviFile || !fb) return;

    uint32_t frameOffset = aviFile.position() - moviOffset - 4;
    uint32_t paddedSize = (fb->len + 1) & ~1;  // RIFF 要求偶數對齊

    // 寫 chunk header
    aviFile.write("00dc", 4);
    aviFile.write((uint8_t*)&fb->len, 4);
    aviFile.write(fb->buf, fb->len);
    if (fb->len & 1) aviFile.write((uint8_t)0);  // padding

    // 記錄索引
    if (frameCount < 30000) {
        idxBuffer[frameCount].offset = frameOffset;
        idxBuffer[frameCount].size   = fb->len;
    }
    frameCount++;
}

void stopRecording() {
    if (!aviFile) return;

    // 回補 LIST 'movi' 大小
    uint32_t moviSize = aviFile.position() - moviOffset - 8;
    aviFile.seek(moviOffset + 4);
    aviFile.write((uint8_t*)&moviSize, 4);
    aviFile.seek(0, SeekEnd);

    // 寫入 idx1
    writeIdx1(aviFile, idxBuffer, frameCount);

    // 回補 RIFF 總大小
    uint32_t fileSize = aviFile.position() - 8;
    aviFile.seek(4);
    aviFile.write((uint8_t*)&fileSize, 4);

    aviFile.close();
    free(idxBuffer);
    idxBuffer = nullptr;
}
```

### 串流 + 錄影同步邏輯

```cpp
void streamTask(void *pvParam) {
    while (true) {
        camera_fb_t *fb = esp_camera_fb_get();
        if (!fb) { vTaskDelay(10); continue; }

        // Wi-Fi 串流（永遠執行）
        if (streamActive) sendMJPEGFrame(fb);

        // SD 錄影（僅模式 1 開啟時）
        if (streamRecord) writeFrame(fb);

        esp_camera_fb_return(fb);
        vTaskDelay(1);
    }
}
```
